# AI Coding CLI 从零到一实践课程

> 通过 8 个阶段、20+ 实践项目，从零开始构建一个功能完整的 AI 编程 CLI 工具

## 课程目标

完成本课程后，你将能够：

1. **理解** AI 编程工具的核心架构设计原理
2. **掌握** LLM API 集成、流式处理、工具调用等关键技术
3. **实现** 一个支持多模型、工具系统、会话管理的完整 CLI
4. **具备** 阅读和扩展 Claude Code、Codex CLI 等工具的能力

## 技术栈

- **语言**: TypeScript 5.x
- **运行时**: Node.js 18+ / Bun
- **CLI 框架**: Commander.js / Oclif
- **AI SDK**: OpenAI SDK / Anthropic SDK
- **数据库**: SQLite + Drizzle ORM (可选)
- **终端 UI**: Ink / Ink-spinner / Chalk

## 快速开始

### 环境准备

```bash
# 安装 Node.js 18+ (推荐使用 nvm)
nvm install 18
nvm use 18

# 或安装 Bun (更快的运行时)
curl -fsSL https://bun.sh/install | bash

# 安装依赖管理工具
npm install -g pnpm
```

### 创建项目

```bash
# 克隆或创建项目目录
mkdir mini-ai-cli && cd mini-ai-cli

# 初始化项目
pnpm init

# 安装核心依赖
pnpm add typescript ts-node @types/node
pnpm add commander chalk ink openai @anthropic-ai/sdk

# 安装开发依赖
pnpm add -D tsx nodemon
```

### 运行示例

```bash
# 进入对应章节目录
cd 01-foundation

# 运行示例代码
tsx src/example.ts
```

## 学习建议

### 核心原则：教程文档驱动

本项目的核心价值是**教程文档**，而非现成的代码实现。学习目标是通过文档一步步自己实现一个功能完整的 AI Coding CLI。

```
┌─────────────────────────────────────────────────────────────────┐
│                      学习理念                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   文档是核心 → 代码是示例 → 动手实现是目标                        │
│                                                                 │
│   • 教程需要详细、循序渐进                                        │
│   • 代码作为参考实现，不要过度工程化                               │
│   • 让学习者能按文档从零实现一个 mini CLI                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 推荐学习顺序

1. **按顺序学习**: 章节之间有依赖关系，建议按顺序学习
2. **动手实践**: 每个章节都有实践项目，务必亲自实现
3. **理解原理**: 不仅要写代码，还要理解背后的设计原理
4. **参考源码**: 结合 Claude Code、Codex CLI 等开源项目的源码学习

### 每章学习流程

```
1. 阅读章节 README → 理解核心概念
2. 运行示例代码 → 观察实际效果
3. 完成练习题 → 巩固知识点
4. 实现实践项目 → 综合应用
5. 对比源码实现 → 深入理解
```

### 教程文档要求

每节教程应包含：

| 部分 | 内容 | 目的 |
|------|------|------|
| 学习目标 | 明确本节要掌握的技能 | 学习者知道要学什么 |
| 代码详解 | 每行代码的注释和解释 | 理解代码背后的原理 |
| 参数说明 | 表格形式解释参数含义 | 快速查阅和参考 |
| 使用示例 | 实际运行的代码示例 | 边学边练 |
| 练习题 | 动手实践的任务 | 巩固知识点 |
| 下一步 | 链接到下一节 | 连贯学习 |

### 关键术语

| 术语 | 说明 | 示例 |
|------|------|------|
| **Provider** | AI 模型提供者的抽象 | OpenAI, Anthropic |
| **Tool** | AI 可调用的工具函数 | read_file, write_file |
| **Session** | 一次完整的对话会话 | 包含多条消息 |
| **Stream** | 流式响应处理 | 实时显示 AI 输出 |
| **Context** | 上下文窗口 | 模型可用的 token 数 |
| **MCP** | Model Context Protocol | 工具集成协议 |
| **Skill** | 技能系统 | 自定义命令 |
